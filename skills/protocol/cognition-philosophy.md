# Cairn Cognition Philosophy

## Projects Are Path-Dependent Cognitive Organisms

A codebase is not a static artifact. It is the accumulated result of thousands of decisions, each constrained by the decisions that came before.

Every project carries invisible knowledge:

- Why certain architectures were chosen over alternatives
- Which approaches were tried and failed
- What constraints exist that are not written in code
- What historical incidents shaped current patterns

This knowledge lives in the heads of team members. When they leave, it leaves with them. When AI tools join, they arrive with no memory of this history.

## Why Passive Memory Fails

Storing decisions in documents and hoping AI reads them is passive memory. It does not work because:

1. AI tools do not reliably read long instruction files
2. Static documents cannot challenge new proposals against historical context
3. Documents cannot detect when their own assumptions have drifted
4. No document can enforce a lifecycle — it can only describe one

## Cognitive Thermodynamics

Cairn treats project knowledge as a thermodynamic system:

**Blood** — active cognition. Decisions, rejections, constraints, and trauma that influence every interaction. Blood events have gravity (G0-G3) that determines their force.

**DNA** — emergent identity. Patterns that emerge from repeated blood events. DNA modulates how new signals are routed and challenged. It is the project's personality.

**Decay** — entropy. Knowledge that is not reinforced fades. Events pass through `ok -> stale -> archived`. This is not a bug — it prevents ancient decisions from permanently blocking evolution.

**Resurrection** — relevance signal. When archived knowledge is repeatedly triggered by new contexts, it resurfaces. The system recognizes that "forgotten" knowledge may still be load-bearing.

**Compression** — emergence. When enough blood events share patterns, they compress into DNA traits. Individual decisions become project identity.

## The Lifecycle Is The Product

Cairn's value is not in its data structures or storage format. It is in the lifecycle:

```
context -> plan -> signal -> observe -> session_end
```

Each step serves a purpose:

- **context** loads constraints before reasoning begins (preventive)
- **plan** challenges direction before implementation starts (diagnostic)
- **signal** captures decisions as they happen (observational)
- **observe** catches implicit decisions before commit (reflective)
- **session_end** runs maintenance and surfaces drift (homeostatic)

Without the lifecycle, Cairn is just YAML files in a directory.

## Governance: Trust But Verify

Not all cognition is equal:

- `system_validated` — low gravity, auto-confirmed (the project trusts AI judgment for small signals)
- `human_ratified` — high gravity, requires human review (the project reserves judgment for significant changes)

This two-tier system means AI can capture routine signals without interrupting workflow, while ensuring that transformative decisions always involve a human.

## Evolution Over Dogma

Historical cognition should inform, not imprison.

The system explicitly supports paradigm shifts through:

- DNA `reevaluation_mode` that suspends trait enforcement
- Decay that naturally reduces the force of old decisions
- Staged governance that requires human ratification for significant changes
- `revisit_when` conditions on accepted debt

A project that cannot evolve is a project that is dying.

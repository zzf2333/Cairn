# Stability

> In 0.x, every contract is best-effort. In 1.x, the Stable region below is SemVer-protected. This page draws the line in advance so the line doesn't have to be argued over later.

## Experimental (may change inside 1.x)

These can change with prominent CHANGELOG callouts. Not SemVer-protected.

- **DNA trait name set.** Today: `simplicity_bias` and `infra_aggressiveness`. More trait kinds planned for 0.5+. Existing trait semantics may also refine.
- **Decay / Resurrection thresholds.** Constants in `cli/src/constants.ts.COGNITIVE_MODE_PARAMS`. Long-horizon dogfood data may justify tuning.
- **Governance policy format.** `governance/policy.yaml` may grow per-tool quota fields.
- **Calibration signal types.** `no_go_violation | skeleton_drift | debt_resolution | dna_drift` ŌĆö additions possible.

---

## Internal (no commitments, change at will)

Free to change. Not breaking even if removed.

- `.cairn/views/` markdown layout ŌĆö auto-generated by `ViewsEngine`. Consumers should use CLI output (`--json`), not parse `views/`.
- `.cairn/sessions/<id>.yaml` internal field layout.
- `.cairn/staged/<id>.yaml` `routing_reason` text content (the field stays; its prose is internal).
- `.cairn/governance/audit.yaml` entry format inside the array.
- `.cairn/logs/tools-*.jsonl` field set.
- TrustRouter internal routing path (the order of the 9 steps is stable; individual implementation is not).
- Any `src/` internal API ŌĆö Store / Engine class signatures.

---

## After 1.0

The Stable region above takes effect with three commitments:

1. **Six-month non-breaking promise.** Any breaking change in the Stable region requires a major bump *and* Ōēź 6 months of public deprecation notice.
2. **`cairn migrate` is mandatory** for every breaking schema change.
3. **Deprecation window** of Ōēź 1 minor version before any tool signature change.

Until 1.0, these are best-effort but not strict.

## Out of scope

Things explicitly not covered by stability commitments at any phase:

- Skill protocol (`skills/cairn/SKILL.md` and `skills/cairn/protocol/`) ŌĆö these are the protocol; they iterate.
- Test fixtures, scenarios, internal benchmarks.
- Anything in `CLAUDE.md`, `docs/internal/`, plan files.
